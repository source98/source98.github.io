    深入理解原型与原型链

    Zeal 6/18/2024 JavaScript原型&原型链

    # 什么是原型，什么是原型链

    在了解原型之前，我们先了解另外一个问题。什么是构造函数？
    其实在 JS 中所有函数都是构造函数，为了方便区分，我们通常把构造函数名首字母大写，如：

    function Person() {
  console.log("我是一个构造函数")
}

    # 构造函数的作用是什么呢？

    构造函数的主要作用是在实例化对象时，定义和设置对象的初始状态。它负责将对象从一种原始状态转变为一个具有特定属性和准备就绪的状态。最主要的是构造函数能帮我们批量生成对象，避免了枯燥的手动创建，经典例子是，白种人、黄种人、黑人都属于人类，只是各自有些不同的特征，我们可以通过人这个构造函数去批量生成各种人种

    function Person(name,age,skinColor){
  this.name - name;
  this.age = age;
  this.skinColor = skinColor;
}

const y = new Person('小明'，18，yellow);
const w = new Person('jack'，20，white);
const b = new Person('小明'，18，black);

console.log(y)
console.log(w)
console.log(b)

    # 什么是原型呢？

    JS 构造函数用来实例化对象时，会在内存中创建一个对应的对象，这个对象就是原型。构造函数默认有一个prototype属性，它的值指向函数的原型同时也可以称呼它为原型对象。原型中有一个constructor属性,它的值指向函数对象。 通过构造函数实例化出来的对象，并不具有原型对象，其默认有一个__proto__属性我们称呼它为隐式原型,它的值指向构造函数的原型。在原型对象上添加或修改的属性，在所有实例化出的对象上都可共享。

    function Person(){}
const p = new Person();

console.log(p.__proto__ === Person.prototype) // true
console.log(p.constructor) // Person();

    # 什么是原型链？

    查找对象实例的方法和属性时，先在自身找，找不到则沿着__proto__向上查找，我们把__proto__形成的链条关系称原型链，实现了js继承。

    根据上面这张图，我通过一个简单的实例来表达原型链的真正含义

    function Foo(){}
const obj = new Foo();
obj = {
  name:'jack',
  age:18
}
Object.prototype.sex = 'male'
obj.__proto__ => Foo.prototype // Foo.prototype.sex => undefined
Foo.prototype.__proto__ => Object.prototype // Object.prototype.sex = male
console.log(obj.sex) // male

    下面有一道简单的题目，可以帮助我们更好的理解原型与原型链

    function Person() {};

Object.prototype.a = 1;
Function.prototype.b = 2;

const p = new Person();

p.a // 1
p.b // undefined
Person.a // 1
Person.b // 2

    为什么p.b的值是undefined?为什么Person.a和Person.b都有值呢？

    首先我肯定是可以知道p是一个实例化对象，当p上没有这个属性的时候我们回去原型上找，p的原型是Person.prototype，此时原型上也没有我们需要继续跟随原型链查找。 Person.prototype.proto 指向了 Object.Prototype,所以我们找到了a属性，b属性是找不到的，因此为undefined； 我们知道Person是个构造函数，但同时它也是个对象，我们依旧可以跟随原型链查找Person.proto => Function.prototype，此时只找到了b属性，我们接着往下找。 Function.prototype的原型指向了Object.Prototype至此a、b属性都找到了。
