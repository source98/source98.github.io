    一文读懂Decorator装饰器

    Zeal 7/14/2024 JavaScript装饰器

    1. 装饰器的样子

    我们首先先来看看Decorator装饰器长什么样子，以下代码摘录自Nest官方文档 (opens new window)

    /*Nest.Js cats.controller.ts */
import {Controller, Get} from '@nestjs/common'

@Controller('cats')
export class CatsController{
  @Get()
  findAll(): string{
    return 'This action returns all cats';
  }
}

    我们先不急着去理解以上代码，先有一个对装饰器的初步映像。

    2. 为什么要理解装饰器

    Decorator 装饰器是 ECMAScript 的语言提案，目前还处于 stage-2 阶段，但是借助 TypeScript 或者 Babel，已经有大量的优秀开源项目深度用上它了，比如：VS Code, Angular, Nest.Js(后端 Node.js 框架), TypeORM, Mobx(5) 等等。了解了装饰器，我们可以学习下一些优质的编码方式以及思想概念。

    3. 装饰器的概念区分

    在了解装饰器之前，有必要先对装饰器的3个概念进行区分。

    1. Decorator Pattern 装饰器模式，是一种抽象的设计理念，核心思想是在不修复原有代码的情况下对功能进行扩展。
    2. Decorator 装饰器，是一种装饰类函数，是对装饰器模式的具体实现。
    3. @Decorator 是装饰器的语法糖。

    4. 装饰器实战

    现在有一个Person类， 它有两个函数方法：eat和play。

    class Person {
  eat(){
    console.log('在吃饭');
  }
  play(){
    console.log('在玩游戏')
  }
}

    现在，我想记录下这个类每个函数的执行时间，方便后期的性能优化。

    4.1 普通做法

    入侵式写入代码

    class Person {
  eat(){
    const start = new Date();
    console.log('在吃饭');
    const end = new Date();
    console.log(`运行时间${end - start}ms`)
  }
  play(){
    const start = new Date();
    console.log('在玩游戏');
    const end = new Date();
    console.log(`运行时间${end - start}ms`)
  }
}

    这样做明显是提高了代码可读性以及维护成本。

    4.2 装饰器模式(重点)

    我们利用装饰器模式的设计理念，用纯函数的形式写一个装饰器，实现记录函数的耗时。

    // 装饰器函数
function decoratorLogTime(target, key) {
  const targetPrototype = target.prototype
  // 1 备份原来类构造器上的属性描述符 Descriptor
  const oldDescriptor = Object.getOwnPropertyDescriptor(targetPrototype, key)

  // 2 编写装饰器函数业务逻辑代码
  const logTime = function (...arg) {
    // Before 钩子
    let start = new Date()
    try {
      // 执行原来函数
      return oldDescriptor.value.apply(this, arg) // 调用之前的函数
    } finally {
      // After 钩子
      let end = new Date()
      console.log(`耗时: ${end - start}ms`)
    }
  }

  // 3 将装饰器覆盖原来的属性描述符的 value
  Object.defineProperty(targetPrototype, key, {
    ...oldDescriptor,
    value: logTime
  })
}


class Person {
  eat() {
    console.log('在吃饭')
  }
  play() {
    console.log('在游戏')
  }
}
// 4 手动执行装饰器函数，装饰 GuanYu 的 attack 函数
decoratorLogTime(Person, 'eat')
// Step4 手动执行装饰器函数，装饰 GuanYu 的 run 函数
decoratorLogTime(Person, 'play')

const person = new Person()
person.eat()
// 耗时: 5ms
person.play()
// 耗时: 0ms

    在理解上面代码之前，我们需要先理解Object的两个方法：Object.getOwnPropertyDescriptor & Object.defineProperty (opens new window) 以上就是装饰器的具体实现方法，其核心思想是：

    • 1.备份原来类构造器的属性描述符，利用Object.getOwnProPertyDescriptor获取。
    • 2.编写装饰器函数业务逻辑代码，添加耗时统计逻辑。
    • 3.用装饰器函数覆盖原来属性描述符的value，利用Object.defineProperty代理。
    • 4.手动执行装饰器函数，装饰类指定属性。

    5. @Decorator语法糖

    下面我们来学习下语法糖的写法

    // 编写装饰器函数业务逻辑代码
function logTime(target, key, descriptor) {
  const oldMethed = descriptor.value
  const logTime = function (...arg) {
    let start = new Date()
    try {
      // 调用之前的函数
      return oldMethed.apply(this, arg)
    } finally {
      let end = new Date()
      console.log(`耗时: ${end - start}ms`)
    }
  }
  descriptor.value = logTime
  return descriptor
}

class Person {
  // 利用 @ 语法糖装饰指定属性
  @logTime
  eat() {
    console.log('在吃饭')
  }

  // 利用 @ 语法糖装饰指定属性
  @logTime
  play() {
    console.log('在游戏')
  }
}

const person = new Person()
person.eat()
// 在吃饭
// 耗时: 2ms
person.play()
// 在游戏
// 耗时: 3ms
